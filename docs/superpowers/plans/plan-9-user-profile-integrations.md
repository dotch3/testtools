# Plan 9 — User Profile + Integrations (Profile page, OAuth accounts, user credentials, project integration config)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the complete user profile experience and two-layer integration architecture. Users manage their personal profile (name, avatar, theme, locale, password), linked OAuth accounts, and per-user integration credentials (API tokens/PATs). Project admins configure project-level integration settings (URLs, repos, keys). When an action is performed (e.g., create bug on GitHub), the system combines the project config with the acting user's personal credential.

**Architecture:** Two-layer integration model:
1. **User-level credentials** (`USER_INTEGRATION_CREDENTIALS`) -- each user stores their own API token/PAT per integration type, encrypted with `ENCRYPTION_KEY`.
2. **Project-level config** (`INTEGRATIONS`) -- admin/lead configures which tracker, base URL, repository, project key per project. The `credential` column is removed from this table.

**Tech Stack:** TypeScript, Fastify 5, Prisma 6, PostgreSQL 16, Next.js 16.2, shadcn/ui, Tailwind CSS 4, next-intl, Lucide React, Zod

**IMPORTANT:** Next.js 16 has breaking changes. Before writing any frontend code, read the relevant guide in `frontend/node_modules/next/dist/docs/`. Heed deprecation notices.
